## Your Tasks

### 1. Information Gathering
- Scrape/read all content from `../../../../../02-Input/files/HelpDoc-20260126.md `
- Cross-reference with the Excel structure table to understand which content goes where

### 2. Content Distribution Planning
- Map gathered information to the corresponding .md files based on the Excel structure
- Identify which files will be **UPDATED** vs **CREATED**
- Flag any content that doesn't clearly map to existing structure

### 3. Content Transformation (Preview Mode)
- **Audience**: Write for biology graduate students, not CS experts
- **Style**: Reader-oriented, explaining "what" and "why" before "how"
- **Technical simplification**: Replace jargon with plain language, add explanatory context
- **Preservation**: Keep original technical content in markdown comments like `<!-- ORIGINAL: [technical text] -->`

### 4. Apply Standard Page Format

Each page must follow this structure:
```
# [Title]

**[What this page aims to do - subtitle as 1-2 short sentences]**

[Introduction paragraph: big picture context - what is this topic and why does it matter? Help readers understand why they should read this page]

## TLDR

- [Key point 1]
- [Key point 2]
- [Key point 3]
- [Key point 4 - if needed]

[Rest of page content...]
```

**TLDR Guidelines**:
- Target: 3-4 bullet points
- Maximum: 5 bullet points
- **⚠️ FLAG** if you need more than 5 points - this indicates the page has too much content and should be split

### 5. Quality Checks (Preview)
- Review for logical flow between sections
- Check readability for the target audience (bio grad students)
- Ensure consistency in terminology and style across pages
- Verify all original technical details are preserved in comments
