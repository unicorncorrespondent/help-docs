# LinearDesign FAQ


在使用 LinearDesign 进行 mRNA 设计时，常见问题主要集中在计费方式、参数设置和结果解读。以下 FAQ 汇总了高频问题与实操建议，便于快速排查与决策。

## 计费与任务

### Q1: LinearDesign 的计费单位是什么？

按**输出序列条数**计费，而不是按输入序列条数计费。

在 Advance 版本中，如果同一条输入序列配置了多组参数（如多个 `beam size` 和多个 `lambda`），系统会按参数组合生成多条序列，并按生成条数计费。

### Q2: 为什么我只提交 1 条输入，却产生了多条计费记录？

因为一次任务可包含多组参数组合。例如：

- `beam size` 设为 `50`、`100`
- `lambda` 设为 `4`、`10`

则会得到 `2 x 2 = 4` 组组合，对应 4 条输出序列，计费也按 4 条计算。

## 参数设置

### Q3: 如何选择平衡因子 lambda？

可按目标优先级选择：

| 目标 | 推荐 λ 区间 | 说明 |
|------|-------------|------|
| 稳定性优先 | 0-2 | 更偏向优化 MFE |
| 综合平衡 | 2-4 | 平衡 MFE 与 CAI |
| 表达优先 | ≥4 | 更偏向优化 CAI |

建议先用多组 λ 做探索，再依据首轮实验结果缩小区间并微调。

### Q4: 如何选择 beam size？

`beam size` 影响搜索范围与多样性：

- 较大值：更接近全局最优，结果更稳定
- 较小值：多样性更高，但可能牺牲部分指标

对于长度为 `N` 的 RNA，`beam size >= N` 与 `beam size = infinite` 的结果通常接近，不建议同时设置以避免重复计算。

### Q5: 为什么不同参数下结果看起来很相似？

常见原因：

- 参数本身差异较小
- 多组 λ 都处于较大区间（CAI 都接近上限）
- 一组使用 `infinite`，另一组已接近 `N`

若希望提升候选差异性，建议拉开 λ 区间并适当降低部分 beam size。

## 输入与运行

### Q6: 支持哪些输入格式？

支持 FASTA 格式的蛋白序列或 mRNA 序列输入，可通过文本框或 `.txt` 文件上传。建议使用 UTF-8 编码。

### Q7: 序列长度超过限制怎么办？

请先确认所选版本：

- Basic / Plus：蛋白 ≤ 500 aa，mRNA ≤ 1500 nt
- Advance：蛋白 ≤ 2000 aa，mRNA ≤ 6000 nt

超过限制时可切换到更高版本，或先对输入做拆分与重设计。

## 结果解读

### Q8: 如何在多条候选序列中快速筛选？

建议按以下顺序：

1. 先看 CAI 与 MFE 的综合分布
2. 再看 5' 附近是否存在潜在结构阻碍
3. 结合实验目标选择若干代表性候选进入验证

### Q9: MFE 更低一定更好吗？

不一定。更低的 MFE 通常表示更稳定，但结构过稳可能影响翻译过程。建议与 CAI 一起综合判断。

### Q10: 模型结果可以直接用于实验吗？

建议先做小规模体外/细胞实验验证，再进入更大规模实验流程。模型结果适合作为高效筛选起点，不应替代实验验证。
