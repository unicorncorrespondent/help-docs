## 1. API 和 SDK 是什么？

- **API**（**Application Programming Interface**，应用程序编程接口）是一种工具或服务的连接方式，允许不同的软件系统之间进行通信。可以理解为一个“窗口”：通过 API，您可以远程调用 HelixFold3 的能力，而无需安装复杂系统或理解底层实现。
- **SDK**（**Software Development Kit**，软件开发工具包）是一组面向开发者的工具与库。我们提供了基于 API 封装的 SDK，帮助您更轻松地接入 HelixFold3。**使用 SDK 时，通常不需要复杂编程知识，按示例代码即可完成结构预测流程。**

一句话总结：

- 有进阶需求的用户可直接使用 API。
- 希望快速上手的用户建议优先使用 SDK。

## 2. 为什么使用 API 或 SDK？

使用 API 或 SDK，您可以：

- **快速调用**：无需配置复杂运行环境，通过网络即可访问 HelixFold3 功能。
- **自动化处理**：结合脚本，实现批量与流水线任务处理。
- **无缝集成**：便于接入现有工具链或应用程序（如分子建模与数据分析平台）。
- **远程使用**：只需网络连接，减少本地部署资源消耗。
- **灵活扩展**：可在 SDK 基础上定制，或直接使用底层 API 开发高级功能。

## 3. 适合哪些场景使用 API？

典型场景包括：

1. **批量任务处理**：对大量分子序列进行结构预测，例如药物筛选、抗体设计、小蛋白/多肽筛选、酶设计。
2. **集成化工作流**：在生物信息或药物研发管线中加入 HelixFold3 模块。
3. **定制化开发**：构建自动化建模平台、可视化工具等复杂应用。
