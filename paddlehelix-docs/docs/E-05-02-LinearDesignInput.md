## FASTA 格式说明

FASTA 格式是生物序列的标准存储格式，LinearDesign 要求输入序列符合 FASTA 标准。了解并正确使用 FASTA 格式对于确保数据被正确解析至关重要。

### 格式结构

```
>序列名称（可选）
序列内容（字母序列）
```

### 各部分说明

| 元素 | 说明 | 示例 |
|------|------|------|
| `>` | 序列标识符起始符号 | `>` |
| 序列名称 | 序列的描述名称（可选） | `my_protein` |
| 换行符 | 标识行与序列行之间的分隔 | - |
| 序列行 |实际的氨基酸或核苷酸序列 | `MKWVTFISLLFLFSSAYS` |

### 输入示例

**蛋白质序列示例：**
```
>METHionine_Protein
MKWVTFISLLFLFSSAYS
```

**mRNA 序列示例：**
```
>mRNA_sequence
AUGUAA...
```

### 格式要求

| 项目 | 蛋白质序列 | mRNA 序列 |
|------|------------|-----------|
| 有效字符 | 20 种标准氨基酸字母 | A、U、C、G |
| 起始要求 | 无特殊要求 | 必须以 AUG 开头 |
| 长度要求 | 必须为 3 的倍数 | 无特殊要求 |

## 版本与长度限制

不同版本的 LinearDesign 支持不同的序列长度上限：

| 版本 | 蛋白质序列 | mRNA 序列 |
|------|------------|-----------|
| Basic | ≤ 500 个氨基酸 | ≤ 1500 个核苷酸 |
| Plus | ≤ 500 个氨基酸 | ≤ 1500 个核苷酸 |
| Advance | ≤ 2000 个氨基酸 | ≤ 6000 个核苷酸 |

:::tip
**长度选择建议：**
- 短肽序列（<100 aa）：推荐使用 Basic 或 Plus 版本
- 中等长度序列（100-500 aa）：根据参数需求选择 Plus 或 Advance
- 长序列（>500 aa）：必须使用 Advance 版本
:::

## 序列验证

上传或输入序列后，系统会自动进行格式验证，确保数据符合设计要求：

| 验证项目 | 说明 |
|----------|------|
| 字符验证 | 检查序列是否包含有效字符 |
| 长度验证 | 检查序列长度是否符合版本限制 |
| 格式验证 | 检查 mRNA 序列是否为 3 的倍数 |
| 起始验证 | 检查 mRNA 序列是否以 AUG 开头 |

**验证失败处理：**
:::tip
- 系统会显示具体的错误信息
- 根据提示修改序列后重新提交
- 常见问题可参考常见问题页面
:::
